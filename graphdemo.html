Sebelum mulai ngoding, lo kirim form brief ke klien. Tujuannya: dapetin semua info yang lo butuhin sebelum mulai, bukan di tengah jalan. Kalau info belum lengkap, proyek belum mulai.

Isi brief yang perlu lo minta:

// Checklist brief klien
// Identity
nama bisnis        "Toko Baju Siti"
tagline            "Baju Muslim Terbaik Tangerang"
deskripsi bisnis   max 160 karakter

// Kontak
nomor WhatsApp     format internasional
instagram          handle @
email              opsional
alamat             kalau ada toko fisik

// Visual
logo               PNG transparan, min 400px
foto hero          landscape, min 1200x600px
foto produk        min 3, rasio sama semua
warna brand        hex code atau referensi

// Konten
headline utama     max 8 kata
layanan/produk     nama + deskripsi singkat
testimoni          min 3, nama + isi

Setelah brief diisi klien, lo terima semua asetnya. Standarkan format gambar sebelum masuk ke repo — ukuran besar = build lebih lambat, website lebih berat.

Standar aset Titraloka:

# Konversi semua gambar ke WebP
# (bisa pakai tools online atau Sharp CLI)

hero.jpg     → hero.webp       max 1200x600, ~150KB
logo.png     → logo.webp       transparan, max 400px
produk-*.jpg → produk-*.webp   rasio 1:1, max 600x600
og-image.jpg → og-image.webp  1200x630, untuk preview link

# Taruh semua di:
public/
  images/
    hero.webp
    logo.webp
    og-image.webp
    products/
      produk-1.webp
      produk-2.webp

Jangan fork di GitHub UI — clone manual biar bisa set remote ke repo baru yang bersih, tanpa history yang nyambung ke template.

# 1. Clone template ke folder nama klien
git clone https://github.com/titraloka/template tokobajusiti
cd tokobajusiti

# 2. Hapus remote lama, ganti ke repo baru klien
git remote remove origin
git remote add origin https://github.com/titraloka/tokobajusiti

# 3. Install dependencies
npm install

# 4. Push initial commit ke repo baru
git push -u origin main

Bikin repo baru di GitHub dulu sebelum step ini. Nama repo: klien-[namabisnis] biar gampang dicari kalau udah banyak.

Ada 3 hal yang dikustomisasi per klien, urutannya penting:

1. Isi client.config.js (udah dijelasin di step 3)

2. Warna brand masuk ke Tailwind:

// tailwind.config.js
import { client } from './content/client.config.js'

export default {
  content: ['./app/**/*.jsx', './components/**/*.jsx'],
  theme: {
    extend: {
      colors: {
        primary: client.brand.primary,
        secondary: client.brand.secondary,
      },
    },
  },
}

Setelah ini, semua component yang pakai bg-primary atau text-primary otomatis pakai warna brand klien.

3. Aktifkan/nonaktifkan sections:

// app/page.jsx
import Hero from '@/components/sections/Hero'
import Services from '@/components/sections/Services'
import Testimonials from '@/components/sections/Testimonials'
import Contact from '@/components/sections/Contact'

// Klien ini ga butuh Gallery — comment aja
// import Gallery from '@/components/sections/Gallery'

export default function Home() {
  return (
    <main>
      <Hero />
      <Services />
      <Testimonials />
      <Contact />
    </main>
  )
}

Sebelum nunjukin ke klien, cek checklist ini di localhost:

# Jalankan dev server
npm run dev
# Buka http://localhost:3000

Checklist preview:
✓ Semua teks dari config terbaca dengan benar
✓ Logo dan gambar tampil, tidak broken
✓ Warna brand sesuai
✓ Tombol WA mengarah ke nomor yang benar
✓ Mobile view (Chrome DevTools → toggle device)
✓ Tidak ada console error

Kalau semua oke, baru kirim preview ke klien. Bisa deploy dulu ke Vercel dengan subdomain sementara seperti tokobajusiti.vercel.app sebelum domain asli dipasang.

Batasi revisi dari awal — kasih tau klien di awal: "2 kali revisi termasuk di harga, lebih dari itu ada biaya tambahan." Ini bukan soal pelit, tapi soal scope management.

Untuk request revisi yang masuk:

# Mayoritas revisi cukup edit client.config.js
# Contoh: klien minta ganti headline

// Sebelum
headline: "Tampil Cantik dengan Baju Muslim Pilihan",

// Sesudah
headline: "Koleksi Baju Muslim Premium untuk Keluarga",

# Save → browser auto refresh → cek → push

Kalau revisinya minta perubahan layout atau section baru — itu di luar scope, masuk ke paket berbeda.

Sekali Vercel udah di-connect ke GitHub repo, setiap push otomatis trigger build dan deploy. Lo ga perlu buka dashboard Vercel sama sekali.

# Commit dan push semua perubahan
git add .
git commit -m "update: konten final klien tokobajusiti"
git push

# Vercel otomatis:
# 1. Detect push
# 2. Run next build
# 3. Deploy ke CDN global
# 4. Live dalam ~1-2 menit

Lo bisa pantau status build di dashboard Vercel atau dari notifikasi email. Kalau build gagal, Vercel kasih log errornya.

Domain dibeli di Niagahoster/Dewaweb, tapi DNS-nya diarahkan ke Vercel. Ada dua cara:

# Cara 1: Ganti Nameserver (lebih simpel)
# Di Niagahoster, ganti NS ke:
ns1.vercel-dns.com
ns2.vercel-dns.com

# Cara 2: Tambah CNAME record (kalau NS ga bisa diganti)
# Di DNS manager Niagahoster, tambah:
Type: CNAME
Name: www
Value: cname.vercel-dns.com

# Di Vercel dashboard:
# Settings → Domains → Add domain → tokobajusiti.com
# Vercel kasih instruksi step by step + HTTPS otomatis

Propagasi DNS biasanya 15-30 menit, kadang sampai 24 jam. Selama nunggu, website tetap bisa diakses via *.vercel.app.

Setelah domain live, langsung setup Google Search Console untuk klien. Ini yang kasih tau Google "website ini ada, tolong crawl."

# Step-by-step:
1. Buka search.google.com/search-console
2. Add property → masukkan domain klien
3. Verify ownership → pilih metode DNS record
   (tambah TXT record di Niagahoster)
4. Setelah verified → Sitemaps
5. Submit: https://tokobajusiti.com/sitemap.xml

# sitemap.xml otomatis di-generate next-sitemap
# setelah next build (via postbuild script)

Google biasanya mulai index dalam 1-7 hari. Ini bisa lo masukin sebagai value-add ke klien — "website lo udah didaftarin ke Google."

Flow update konten yang efisien — target under 10 menit dari request masuk sampai live:

# Klien WA: "Kak bisa ganti foto produk yang ini?"

1. Klien kirim foto baru via WA/Drive
2. Lo compress + konversi ke WebP
3. Replace file di public/images/
4. git add . && git commit -m "update: foto produk"
5. git push
6. Tunggu ~2 menit → live otomatis

# Untuk ganti teks/info:
1. Edit client.config.js
2. git add . && git commit -m "update: info kontak"
3. git push → live

Ini bisa dijadiin layanan tambahan — "maintenance bulanan" dengan harga flat. Misalnya Rp150-300rb/bulan untuk update konten sampai X kali.

Hal-hal yang perlu dicek secara berkala:

# Tahunan
- Renewal domain klien (ingatkan 1 bulan sebelum expired)
- Cek Google Search Console — ada error crawl?

# Per 3 bulan
- Update dependencies Next.js
  npm update
  npm audit fix

# Kalau ada masalah
- Website down? Cek Vercel dashboard (status build)
- Domain ga bisa diakses? Cek DNS di Niagahoster
- Build error? Lihat log di Vercel → biasanya
  dependency conflict atau breaking change

Renewal domain bisa lo jadiin reminder otomatis di Google Calendar — set 45 hari sebelum expired untuk semua klien. Kalau lo yang pegang domain klien, ini jadi recurring revenue juga.
